java.util
ArrayList<E>
Lista dinámica que puede crecer o reducirse. E es el tipo de elementos que contiene.
Importación necesaria: import java.util.ArrayList;
⚠ Regla de oro: los métodos se llaman sobre el objeto ArrayList con el punto.
Ejemplo: si tu lista se llama animales → animales.add(m), nunca m.add(animales).

Patrones de herencia
PatrónSintaxis · Cuándo usarlo
extends Herencia. La subclase hereda atributos y métodos del padre. public class Perro extends Mascota { ... }
super(params) Llama al constructor del padre. Siempre primera línea del constructor hijo. public Perro(String nombre, int edad, String raza) { super(nombre, edad); // inicializa Mascota this.raza = raza; }
super.metodo() Llama al método del padre desde la subclase. Usar cuando quieres añadir comportamiento encima del padre. @Override public void atacar() { super.atacar(); // ejecuta el del padre primero System.out.println(getNombre() + " lanza hechizo"); }
@Override Indica que el método sobrescribe al del padre. El compilador verifica que el método exista en el padre. // Con super.metodo() → añades comportamiento al padre // Sin super.metodo() → reemplazas completamente el comportamiento
abstract class Clase que no se puede instanciar directamente. Existe para ser heredada. public abstract class Mascota { ... } // new Mascota() → ❌ error // new Perro() → ✅ correcto
abstract void metodo() Método sin cuerpo. Obliga a todas las subclases concretas a implementarlo. public abstract void volar(); // sin llaves, sin cuerpo
Polimorfismo — regla clave
ElementoSe resuelve enEjemplo
Métodos (@Override) Runtime — usa el tipo real del objeto Mascota m = new Perro("Toby", 3); m.muestra(); // ejecuta muestra() de Perro, no de Mascota
Atributos Compilación — usa el tipo declarado // ❌ NUNCA redeclares atributos en subclases → ocultación // ✅ Usa private + getter en el padre

Conceptos POO — RA6
Genéricos <T>
Permiten crear clases y métodos que funcionan con cualquier tipo, sin duplicar código.
T es un placeholder — se sustituye por el tipo real al instanciar.
⚠ Usar cuando la estructura o lógica es idéntica independientemente del tipo de dato.
Patrones de genéricos
PatrónSintaxis · Ejemplo
Clase genérica Declarar la clase con <T> después del nombre. public class Caja<T> { private T contenido; public Caja(T contenido) { this.contenido = contenido; } public T getContenido() { return contenido; } }
Instanciar clase genérica Especificar el tipo entre <> al crear el objeto. Caja<Documento> caja1 = new Caja<>(miDocumento); Caja<Electronica> caja2 = new Caja<>(miElectronica);
ArrayList<T> interno Clase genérica con colección interna — patrón Repositorio. public class Repositorio<T> { private ArrayList<T> elementos = new ArrayList<>(); public void añadir(T elemento) { elementos.add(elemento); } public void eliminar(T elemento) { elementos.remove(elemento); } public int total() { return elementos.size(); } }

java.lang — RA3
Manejo de Excepciones
Estructura try-catch-finally
BloquePara qué sirve · Ejemplo
try { } Código que puede lanzar una excepción. try { int resultado = 10 / divisor; }
catch (TipoEx e) { } Se ejecuta si se lanza la excepción indicada. e.getMessage() devuelve el mensaje original. catch (ArithmeticException e) { System.out.println("No se puede dividir entre cero"); System.err.println(e.getMessage()); }
finally { } Se ejecuta siempre, haya o no excepción. Opcional. finally { System.out.println("Operación finalizada"); }
Excepciones más comunes
ExcepciónCuándo se lanza · Ejemplo
ArithmeticException División entre cero con enteros. int r = 10 / 0; // lanza ArithmeticException
NumberFormatException Integer.parseInt() con un String no numérico. int n = Integer.parseInt("abc"); // lanza NumberFormatException
ArrayIndexOutOfBoundsException Acceso a posición que no existe en un array. int[] v = {1, 2, 3}; int x = v[5]; // lanza ArrayIndexOutOfBoundsException
NullPointerException Llamar a un método sobre un objeto que es null. String s = null; s.length(); // lanza NullPointerException
Excepción personalizada
PatrónSintaxis · Ejemplo
extends Exception Crear una excepción propia heredando de Exception. public class IPInvalidaException extends Exception { public IPInvalidaException(String mensaje) { super(mensaje); } } // Lanzarla: throw new IPInvalidaException("Formato IP incorrecto"); // Capturarla: catch (IPInvalidaException e) { System.err.println(e.getMessage()); }

java.util.regex
Expresiones Regulares — Pattern · Matcher
Importaciones: import java.util.regex.Pattern; · import java.util.regex.Matcher;
Uso básico
PatrónSintaxis · Ejemplo
Validar si coincide String regex = "ALM-[0-9]{4}"; if (texto.matches(regex)) { System.out.println("Válido"); }
Buscar dentro de texto Pattern p = Pattern.compile("[A-Z]{3}-[0-9]{4}"); Matcher m = p.matcher(texto); while (m.find()) { System.out.println("Encontrado: " + m.group()); }

⚠ Problema frecuente: después de nextInt() o nextDouble() siempre hay un salto de línea pendiente. Si a continuación usas nextLine(), añade un sc.nextLine() extra para consumirlo.
Method Summary
Firma del métodoQué hace · Ejemplo
new Scanner(System.in) Crea el lector de teclado. Hacerlo una sola vez. Scanner sc = new Scanner(System.in);
String nextLine()String Lee una línea completa (con espacios). System.out.print("Nombre: "); String nombre = sc.nextLine();
int nextInt()int Lee un entero. Lanza InputMismatchException si no es entero. System.out.print("Edad: "); int edad = sc.nextInt();
double nextDouble()double Lee un decimal. double precio = sc.nextDouble();
